# 網站內容維護指南

這份文件說明如何用瀏覽器上的後台介面（CMS）維護網站內容——**不需要寫程式、也不用直接改 JSON**。

適用對象：行銷／PM 等非工程夥伴。看不懂的地方隨時找工程師，不要硬猜。

> **開始之前**：使用前請先請工程師開通你的編輯權限。

## 先讀：三個最重要的原則

1. **後台不會直接動到正式網站**：你的每次修改都會自動送出成一個「待審核的更新」，由工程師確認後才上線——所以放心編輯。
2. **等檢查綠勾**：送出後系統會自動檢查格式。綠勾＝沒問題；紅叉＝有錯（多半是必填漏填），點進去看訊息或找工程師。
3. **由工程師確認上線**：綠勾後請通知工程師確認合併，約 2–3 分鐘自動上線。

## 怎麼進入後台

1. 開啟 **<https://kiwissec.github.io/admin/>**。
2. 點 **Sign in with GitHub**，用已被授權的帳號登入（沒有權限請先找工程師開通）。
3. 登入後左側會看到四個內容區：**課程、服務、最新消息、學員見證**。

## 怎麼編輯（後台操作）

### A. 修改現有內容（最常見）

1. 點左側的內容區（例如「最新消息」）。
2. 點要修改的那一筆，進入表單。
3. 直接在欄位裡改文字。下拉、日期等欄位用點選的，不用記格式。
4. 右上角點 **Save**，系統會把這次修改送出待審核。
5. 通知工程師檢視並確認上線。

### B. 新增一筆

1. 進入內容區，點右上角 **New（新增）**。
2. 逐欄填寫（必填欄位會標示）。**`代號 (id)` 必須唯一**。
3. **Save** → 送出待審核。

### C. 刪除一筆

1. 進入該筆資料，於右上角選單點 **Delete**。
2. **Save / 確認** → 送出待審核。

### D. 調整顯示順序

- 課程／服務／見證：改 **顯示順序**（數字小的排前面），維持每筆不同。
- 最新消息：改 **日期**（網站自動依日期由新到舊排序）。

## 各內容區欄位說明

下列欄位名稱與後台表單上的標示一致；後台每個欄位下方也有提示文字。

### 課程（courses）

| 欄位      | 說明                                                                   |
| --------- | ---------------------------------------------------------------------- |
| 代號 (id) | 唯一代號，英數與連字號（例如 `aw-14`）；不可重複                       |
| 顯示順序  | 數字小的排前面，每門課不同                                             |
| 分類      | **下拉選一個**：資安意識／攻防演練／資安防護                           |
| 課程名稱  | 課程標題                                                               |
| 標籤      | 逐項輸入，至少一個                                                     |
| 課程簡介  | 一段說明文字（卡片與內頁開頭都會顯示）                                 |
| 時數      | 文字即可（例如 `1~3 小時`）                                            |
| 課程介紹  | 選填；課程內頁（點課程卡片後進入）開頭的介紹段落，段落之間空一行可分段 |
| 課程大綱  | 選填；逐項輸入，課程內頁以編號清單呈現                                 |
| 適用人員  | 選填；一段文字，說明適合對象                                           |
| 課程目標  | 選填；逐項輸入，課程內頁以清單呈現                                     |

> 「課程介紹／課程大綱／適用人員／課程目標」皆為**選填**：填了會顯示在該門課的內頁（在首頁或課程頁點該課卡片後進入），全部留空則內頁只顯示課程簡介與洽詢按鈕。

### 服務（services）

| 欄位                               | 說明                                                                     |
| ---------------------------------- | ------------------------------------------------------------------------ |
| 代號 (id)                          | 唯一代號（例如 `education`），不可重複                                   |
| 顯示順序                           | 數字小的排前面，每筆不同                                                 |
| 圖示                               | 圖示代碼，一般不用改；要換請找工程師                                     |
| 服務名稱 / 一句話短說明 / 完整說明 | 文字                                                                     |
| 重點清單                           | 逐項輸入，至少一個                                                       |
| 按鈕                               | 可有多顆；每顆填**按鈕文字**與**連結**，並可勾選「主要按鈕」「外部連結」 |

### 最新消息（news）＋ 上傳圖片

| 欄位            | 說明                                                                               |
| --------------- | ---------------------------------------------------------------------------------- |
| 代號 (id)       | 建議用該則日期（例如 `2026-01-31`），不可重複                                      |
| 標籤            | 文字；會變成最新消息頁可篩選的分類                                                 |
| 日期            | 用日期選擇器；網站依日期由新到舊排序                                               |
| 圖片檔名        | **只填檔名**（例如 `260131.webp`），不要填路徑                                     |
| 標題 / 內容說明 | 文字                                                                               |
| 連結網址        | 選填；填了之後點該則標題會另開分頁前往（例如該則 Facebook 貼文），留空則該則不可點 |

**上傳圖片步驟**：先用後台上方的 **媒體庫（Asset Library）** 把圖片上傳，再回到「圖片檔名」欄位填入剛上傳的檔名。建議使用 `.webp` 格式。（若在後台找不到媒體庫，請工程師協助。）

### 學員見證（testimonials）

| 欄位      | 說明                            |
| --------- | ------------------------------- |
| 代號 (id) | 唯一代號（例如 `t7`），不可重複 |
| 顯示順序  | 數字小的排前面，每筆不同        |
| 見證內容  | 學員的話                        |
| 身分      | 例如 `軟體工程師`               |
| 圖示      | 一般不用改                      |

## 送出之後會發生什麼

1. 你按 Save → 後台自動把修改送出成一個「待審核的更新」。
2. 系統自動檢查格式（**綠勾＝沒問題；紅叉＝有錯**）。
3. 工程師看過、確認綠勾後合併。
4. 合併後約 2–3 分鐘，網站自動更新、線上可見。

## 常見問題

- **送出後出現紅叉**：點開檢查訊息看哪個欄位有問題（多半是必填漏填或 `代號 (id)` 重複），修正後再 Save。
- **看不到某個內容區或無法登入**：多半是權限尚未開通，找工程師。
- **改了沒上線**：確認工程師已確認合併你的更新；合併後才會上線。

## 需要協助

不確定能不能改的內容，先問工程師再改，避免上線出錯。
